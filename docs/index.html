Overview

The APB I2C Controller is a compact I2C master IP intended for FPGA or ASIC SoC integration. Software configures the controller through APB registers, then starts a byte-level I2C transaction. The RTL generates the START condition, address phase, data phase, ACK/NACK sampling, and optional STOP condition.

Supported

  • I2C master mode
  • 7-bit slave address
  • Single-byte write transaction
  • Single-byte read transaction
  • Programmable clock divider
  • Done and acknowledge-error interrupts
  • Open-drain SDA/SCL output-enable pins

Planned Extensions

  • Multi-byte FIFO transfers
  • Repeated START sequencing
  • Clock stretching wait states
  • Bus arbitration loss detection
  • 10-bit addressing
  • Full I2C slave BFM verification

I2C Protocol Summary

I2C is a two-wire, open-drain serial bus. The bus has serial data SDA and serial clock SCL. Devices only pull a line low; external pull-up resistors return released lines high.

SignalBehavior
SDASerial data line. Data is stable while SCL is high, except during START and STOP.
SCLSerial clock line generated by the master in this IP.
STARTSDA transitions high-to-low while SCL is high.
STOPSDA transitions low-to-high while SCL is high.
ACKThe receiver pulls SDA low on the ninth clock.
NACKSDA remains high on the ninth clock.

Byte Format

START
7-bit slave address
R/W bit
ACK bit
8-bit data
ACK/NACK bit
STOP

For write transactions, the IP sends the address with R/W = 0, checks the address ACK, sends one byte from TXDATA, checks the data ACK, and optionally emits STOP. For read transactions, the IP sends the address with R/W = 1, checks the address ACK, samples one byte from SDA, sends a NACK to end the read byte, and optionally emits STOP.

Programming Model

Single-Byte Write

  1. Write CTRL.ENABLE = 1.
  2. Write CLKDIV for the desired SCL rate.
  3. Write the 7-bit target address to ADDR[6:0].
  4. Write the payload byte to TXDATA[7:0].
  5. Write CMD = START | STOP, with READ = 0.
  6. Poll STATUS.DONE or wait for IRQ.
  7. Check STATUS.ACK_ERROR or IRQ_STAT[1].
  8. Clear status by writing IRQ_STAT = 0x3 if needed.
// Example write transaction
APB_WRITE(0x00, 0x00000001); // CTRL.ENABLE
APB_WRITE(0x08, 0x00000063); // CLKDIV
APB_WRITE(0x0C, 0x00000050); // ADDR = 0x50
APB_WRITE(0x10, 0x000000A5); // TXDATA
APB_WRITE(0x18, 0x00000005); // START=1, READ=0, STOP=1

Single-Byte Read

  1. Write CTRL.ENABLE = 1.
  2. Write CLKDIV.
  3. Write the 7-bit target address to ADDR[6:0].
  4. Write CMD = START | READ | STOP.
  5. Poll STATUS.DONE or wait for IRQ.
  6. Read RXDATA[7:0].
// Example read transaction
APB_WRITE(0x00, 0x00000001); // CTRL.ENABLE
APB_WRITE(0x08, 0x00000063); // CLKDIV
APB_WRITE(0x0C, 0x00000050); // ADDR = 0x50
APB_WRITE(0x18, 0x00000007); // START=1, READ=1, STOP=1
APB_READ (0x14, rxdata);     // RXDATA[7:0]

I2C Core FSM

StateAction
I2C_ST_IDLECore idle, bus released, waiting for start pulse.
I2C_ST_START_APull SDA low while SCL is released high to create START.
I2C_ST_START_BPull SCL low and prepare for bit shifting.
I2C_ST_SHIFT_LOWDrive or release SDA while SCL is low.
I2C_ST_SHIFT_HIGHRelease SCL high and sample read data bits if in read data phase.
I2C_ST_ACK_LOWPrepare the ACK/NACK cycle.
I2C_ST_ACK_HIGHSample slave ACK for transmitted bytes or emit master NACK after read data.
I2C_ST_STOP_APull SDA low with SCL low before STOP.
I2C_ST_STOP_BRelease SCL and SDA to create STOP, then assert done.

Timing Model

CLKDIV is loaded into an internal down-counter. Each time the counter reaches zero, the I2C FSM advances one timing step and the counter reloads. A full bit uses two main steps: one low phase and one high phase. 

tick_period_cycles = CLKDIV + 1
approx_scl_period  = 2 * tick_period_cycles * pclk_period
approx_scl_hz      = pclk_hz / (2 * (CLKDIV + 1))
This equation describes the current RTL timing model. Production I2C implementations should also check setup/hold margins, rise time from board pull-ups, clock stretching, and target I2C mode requirements.

Integration Notes

  • Use apb_i2c_wrapper as the single top-level module. It exposes b_apb_i2c_sda and b_apb_i2c_scl directly.
  • The local GF180 path provided for this task is gf180mcu_fd_sc_mcu9t5v0, a standard-cell library. When GF180MCU_SC is defined, apb_i2c_io_cell.sv maps the open-drain function to tiel, bufz_4, and buf_2.
  • Run make lint-gf180 to lint the pad wrapper using the local GF180 standard-cell models.
  • Provide board-level pull-up resistors on SDA and SCL.
  • Keep i_apb_i2c_pclk fast enough to generate the required SCL rate using CLKDIV.
  • Do not launch a new command while STATUS.BUSY is 1.
  • Clear IRQ_STAT after servicing interrupts to allow future interrupt edges/status events.
  • For production use, add a pad-level wrapper that maps output-enable polarity to the selected FPGA or ASIC IO cell.
